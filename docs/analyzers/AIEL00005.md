# AIEL00005 - General - Multiple Mediator Dispatch Calls

## Summary

A single method calls the mediator more than once.

## Problem

Each dispatch creates its own DI scope, so repeated calls can hide transaction or coordination issues.

## Solution

Prefer a single command or query, or move shared coordination into a pipeline behavior.

## Why This Matters

Keeping related work in one dispatch makes scope boundaries and cross-cutting behavior easier to reason about.

## Additional Information

- Category: Usage
- Severity: Warning
- Enabled by default: Yes
