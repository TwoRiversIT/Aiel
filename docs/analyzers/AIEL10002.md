# AIEL00014 - Strong ID - Must Not Use Positional Record Syntax

## Summary

Strong ID declarations must not use positional record syntax.

## Problem

The type uses a positional record declaration that leaves the generator unable to own the constructor shape.

## Solution

Use a non-positional partial record declaration for the Strong ID.

## Why This Matters

The generator needs control over the validating constructor and generated members.

## Additional Information

- Category: Aiel.StrongIds.Generators
- Severity: Error
- Enabled by default: Yes
