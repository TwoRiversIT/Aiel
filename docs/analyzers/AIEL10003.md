# AIEL00015 - Strong ID - Must Implement Matching Interface

## Summary

Strong ID declarations must implement IStrongId with the same backing value type.

## Problem

The Strong ID type does not implement the matching interface for the type supplied to the StrongId attribute.

## Solution

Implement IStrongId for the same TValue used by the attribute.

## Why This Matters

The interface ties the generated type to the value semantics expected by the rest of the system.

## Additional Information

- Category: Aiel.StrongIds.Generators
- Severity: Error
- Enabled by default: Yes
