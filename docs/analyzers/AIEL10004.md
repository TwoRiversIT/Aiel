# AIEL00016 - Strong ID - Must Not Declare Value Member

## Summary

Strong ID declarations must not declare their own Value member.

## Problem

The type already defines a Value member that conflicts with the generated member.

## Solution

Remove the custom Value member and let the generator own that API surface.

## Why This Matters

The generator needs a predictable member layout to emit consistent code.

## Additional Information

- Category: Aiel.StrongIds.Generators
- Severity: Error
- Enabled by default: Yes
