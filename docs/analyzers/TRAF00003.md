# TRAF00003: Use ResultHttpClientExtensions for Result types

## Summary
When deserializing `Result` and `Result<T>` types over HTTP, use the specialized extension methods from `ResultHttpClientExtensions` instead of generic `HttpClient` JSON methods. This ensures proper configuration of JSON serialization options and error handling.

## Problem
```csharp
// ❌ Generic method doesn't use Result converters
var result = await client.GetFromJsonAsync<Result<WeatherForecast>>("weather");

// ❌ Manual deserialization bypasses configuration
var json = await client.GetStringAsync("api/data");
var result = JsonSerializer.Deserialize<Result<Data>>(json);

// ❌ Missing error context
var response = await client.PostAsJsonAsync("api/orders", order);
var result = await response.Content.ReadFromJsonAsync<Result<int>>();
```

## Solution
Use `ResultHttpClientExtensions` methods that automatically apply configured JSON options:

```csharp
using Aiel.Results;

public class WeatherApiClient
{
    private readonly HttpClient _httpClient;

    public WeatherApiClient(HttpClient httpClient) => _httpClient = httpClient;

    // ✅ GET request
    public async Task<Result<WeatherForecast>> GetWeatherAsync(string location)
    {
        return await _httpClient.GetResultAsync<WeatherForecast>($"weather/{location}");
    }

    // ✅ POST request
    public async Task<Result<OrderConfirmation>> CreateOrderAsync(CreateOrderRequest request)
    {
        return await _httpClient.PostAndReturnResultAsync<CreateOrderRequest, OrderConfirmation>(
            "orders", 
            request);
    }

    // ✅ PUT request
    public async Task<Result<User>> UpdateUserAsync(int id, UpdateUserRequest request)
    {
        return await _httpClient.PutAndReturnResultAsync<UpdateUserRequest, User>(
            $"users/{id}", 
            request);
    }

    // ✅ PATCH request
    public async Task<Result<User>> PartialUpdateAsync(int id, PatchUserRequest request)
    {
        return await _httpClient.PatchAndReturnResultAsync<PatchUserRequest, User>(
            $"users/{id}", 
            request);
    }

    // ✅ DELETE request
    public async Task<Result<Unit>> DeleteUserAsync(int id)
    {
        return await _httpClient.DeleteAndReturnResultAsync<Unit>($"users/{id}");
    }
}
```

## Available Methods

| Method | Purpose |
|--------|---------|
| `GetResultAsync<T>` | GET request returning `Result<T>` |
| `PostAndReturnResultAsync<TReq, TRes>` | POST with request body, returns `Result<TRes>` |
| `PutAndReturnResultAsync<TReq, TRes>` | PUT with request body, returns `Result<TRes>` |
| `PatchAndReturnResultAsync<TReq, TRes>` | PATCH with request body, returns `Result<TRes>` |
| `DeleteAndReturnResultAsync<T>` | DELETE request returning `Result<T>` |
| `ResultAsync<T>` (on HttpResponseMessage) | Convert response to `Result<T>` |

## Why This Matters

### Proper Configuration
These methods use `Results.JSO`, which is automatically configured during `AddResultPattern()` startup with all necessary converters for polymorphic error deserialization.

### Error Handling
The methods include built-in error handling that extracts error information from failed responses, ensuring you get meaningful error objects instead of null.

### Consistency
Enforces a consistent pattern across your application for handling Result types over HTTP, improving code clarity and maintainability.

### Type Safety
Each method is specifically designed for Result types, providing better intellisense and compile-time checking.

## Setup

Ensure `AddResultPattern()` is called during application startup:

```csharp
// Program.cs
var builder = WebApplicationBuilder.CreateBuilder(args);
builder.Services.AddResultPattern();
var app = builder.Build();
```
